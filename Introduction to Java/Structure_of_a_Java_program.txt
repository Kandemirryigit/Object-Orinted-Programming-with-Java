1-Anatomy of a Java Program

  1-Packages:
  - Packages are used to organize related classes and interfaces into namespaces
  - Think of them as folders in a file system
  - They help prevent naming conflicts and make it easier to manage large projects.

        *Purpose:
        - To organize and group related classes.

        *Syntax:
        - package packageName;
        - placed at the top of the Java file

        *Example:
        - package com.example.myapp;
        - In this example, com.example.myapp is the package name
        - It's a common convention to use reverse domain name notation for package names to ensure uniqueness.


        * Real World Example:
        - In a large e-commerce application, you might have packages like com.ecommerce.products, com.ecommerce.users, and com.ecommerce.orders
        - to organize the code related to products, users, and orders, respectively.

        * Hypothetical Scenario:
        - Imagine you're building a game
        - You could use packages to separate different aspects of the game, such as com.game.graphics, com.game.audio, and com.game.logic.









    2-classes:
    - A class is a blueprint for creating objects.
    - It defines the data (fields) and behavior (methods) that objects of that class will have
    - In Java, everything revolves around classes.


        *Purpose:
        - To define the structure and behavior of objects


        *Syntax:
        - public class ClassName <ClassName>


        *Example:
        package com.example.myapp;

        public class MyClass {
            // Fields (variables)
            int x;
            String name;
            // Methods (functions)
            public void myMethod() {
                System.out.println("Hello from MyClass!");
            }
        }

        - In this example, MyClass is the name of the class
        - It has two fields (x and name) and one method (myMethod)
        - The public keyword means that the class is accessible from anywhere.


    
        *Real World Example:
        - Think of a Car class
        - It would have attributes like color, model, and speed, and methods like accelerate(), brake(), and turn().



        *Hypothetical Scenario:
        - In a school management system, you might have classes like Student, Teacher, and Course, each with its own attributes and methods.








    3-main method:
    - The main method is the entry point of a Java program
    - When you run a Java program, the Java Virtual Machine (JVM) starts executing code from the main method.
    - very standalone Java application must have a main method.

        
        *Purpose:
        - To provide the starting point for program execution.


        *Syntax:
        - public static void main(String[] args) { ... }


        *Explanation:
        - public: Makes the method accessible from outside the class.
        - static: Allows the method to be called without creating an object of the class.
        - void: Indicates that the method doesn't return any value.
        - main: The name of the method, which is required by the JVM.
        - String[] args: An array of strings that allows you to pass arguments to the program from the command line


        * Example:
        package com.example.myapp;

        public class Main {
        public static void main(String[] args) {
            System.out.println("Hello, World!");
        }
        }




        *Real-World Example:
        - In a web server application, the main method might be responsible for starting the server and listening for incoming requests.


        *Hypothetical Scenario:
        - In a data processing application, the main method might read data from a file, process it, and write the results to another file.


    



   4-Statements:
   - Statements are the individual instructions that make up a Java program
   - They tell the computer to perform a specific action.
   - Statements are terminated by a semicolon (;).


        *Purpose:
        -  To perform actions.


        *Example:
        int x = 10; // Declaration and assignment statement
        System.out.println(x); // Output statement
        x = x + 5; // Assignment statement

        - These are simple examples of statements
        - They declare a variable, print its value, and then update its value.



        *Types of Statements:

            * Declaration Statements:
            - Used to declare variables (e.g., int x;).

            * Assignment Statements:
            - Used to assign values to variables (e.g., x = 10;).

            * Control Flow Statements: 
            - Used to control the flow of execution (e.g., if, for, while)

            * Method Call Statements: 
            - Used to call methods (e.g., System.out.println("Hello");)
